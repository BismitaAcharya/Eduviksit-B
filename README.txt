Double-click index.html to open the site in your browser. All pages link to
each other with relative paths, so keep every file in this same folder.
No server or install required.

BEFORE THIS GOES LIVE

1. Legal pages are drafted templates, not finalized documents. Have a lawyer
   review and complete the bracketed placeholders (registered entity name,
   address, grievance officer, governing jurisdiction) before publishing.

2. No fabricated trust signals were added — no invented client counts, years
   of experience, or logos. Add real ones (testimonials, client logos,
   founder bios, office address, phone number) as they become available;
   placeholders are marked clearly in the code (search for "[" or
   "to be inserted").

3. The contact form is a working front-end design but isn't wired to a
   backend yet — it currently shows a confirmation message on submit without
   actually sending anything. Connect it to your email/CRM before launch.

4. Blog page lists 30 planned article titles and summaries; the full
   articles themselves haven't been written yet.

5. This is static HTML/CSS/JS with no CMS. For a content-heavy consultancy
   site, migrating this design into Next.js + a headless CMS (as
   recommended in our earlier tech discussion) is the natural next step
   once content and structure are finalized.

NEED CHANGES
Since everything is plain HTML/CSS in a shared design system (colors and
fonts defined once near the top of each file), swapping colors, copy, or
adding pages is straightforward — just ask.
